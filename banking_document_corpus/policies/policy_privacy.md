## Executive Summary

FirstUK Bank is committed to protecting customer personal data in compliance with GDPR and UK Data Protection Act 2018. This policy establishes procedures for data collection, processing, storage, and customer rights management.

---

## Purpose & Scope

### Purpose

To establish procedures for personal data protection that:
- Comply with GDPR and UK Data Protection Act
- Respect customer privacy rights
- Secure personal data against unauthorized access
- Manage customer subject access requests
- Implement data subject rights (access, erasure, portability)

### Scope

Applies to:
- All customer personal data
- All staff and contractors
- All data processing activities
- All systems and storage

---

## Policy Statement

**FirstUK Bank processes customer personal data only where necessary for banking services, with proper safeguards and customer control.**

### Core Requirements

1. **Lawful Basis:** Data processed only for banking/regulatory purposes
2. **Transparency:** Clear privacy notices provided to customers
3. **Data Security:** Encryption and access controls
4. **Customer Rights:** Subject access requests processed within 30 days
5. **Retention:** Data retained only as long as necessary
6. **Consent Management:** Explicit consent for marketing communications

## Data Subject Rights (GDPR)

### Right of Access

**Customer Right:** Request copy of personal data held

**Process:**
- Written request to privacy@firstuk.bank
- Identity verification (for security)
- Response within 30 days (or 60 days if complex)
- Data provided in electronic format
- No fee (unless excessive requests)

### Right to Erasure ("Right to be Forgotten")

**Conditions:**
- Data no longer necessary for purpose
- No legal obligation to retain
- No legitimate interest in retention
- Customer withdraws consent (if basis)

**Exceptions (Data Retained Even if Requested):**
- Legal/regulatory obligation (AML, KYC records)
- Ongoing disputes or complaints
- Fraud investigation
- Financial claim pending
- Active account or product

### Right to Rectification

**Customer Right:** Correct inaccurate data

**Process:**
- Submit correction request
- FirstUK Bank verifies requested change
- Update systems within 30 days
- Notify third parties if data shared

### Right to Restrict Processing

**Customer Right:** Limit how data used (but not require deletion)

**Scenario Example:**
- Customer disputes accuracy
- Data processing while accuracy under dispute
- Data marked "restricted" in system
- Limited processing only for legal obligations

### Right to Data Portability

**Customer Right:** Obtain data in portable format for transfer to another provider

**Process:**
- Written request specifying data
- Data provided in common electronic format (CSV, JSON)
- Response within 30 days
- Applies to data provided by customer or generated through use

### Right to Object

**Customer Right:** Object to processing

**Scope:**
- Direct marketing: Can opt out (mandatory)
- Profiling/automated decisions: Can object
- Legitimate interest processing: Can object if no compelling reason

**Marketing Opt-Out:**
- Email preference: link in every marketing email
- Phone preference: request to support team
- SMS preference: text "STOP" to number
- Recorded immediately in system

### Right to Not Be Subject to Automated Decisions

**Protection:** High-impact automated decisions require human review

**Examples:**
- Automated lending decision (mortgage application rejection)
- Automated fraud detection (account freeze)
- Automated credit scoring

**Right:**
- Be informed of automated decision
- Request human review
- Provide own perspective
- Appeal decision

## Data Security

### Encryption

**In Transit:**
- TLS 1.3 for all data transmission (web, mobile)
- HTTPS enforced on all customer-facing systems
- VPN for staff remote access

**At Rest:**
- AES-256 encryption for databases
- Encrypted backups (S3, off-site)
- Encrypted file storage
- Encrypted USB/portable devices

### Access Control

**Role-Based Access:**
- Customer-facing staff: Limited to customer data needed for service
- Compliance staff: Full data access for regulatory purposes
- Technology staff: Database access (with audit logging)
- Executives: Aggregated data only (no personal details)

**Authentication:**
- Multi-factor authentication for admin/compliance access
- Username/password + OTP
- Audit logging of all access

### Data Breach Response

**Notification Requirement (GDPR):**
- If breach likely to cause high risk to individuals
- Notify individuals within 72 hours
- Notification includes: What happened, what data, mitigation steps, contact for questions

**Internal Notification:**
- Breach identified → Escalate to Data Protection Officer
- Investigation within 24 hours
- Assessment of risk level
- Notification decision
- Regulatory notification (if required to ICO)

## Subject Access Request (SAR) Process

**Customer Right:** Obtain copy of personal data

**Process:**

1. **Request Submission**
   - Email: privacy@firstuk.bank
   - Phone: 0800-555-2265
   - In person: Any branch

2. **Verification**
   - FirstUK Bank verifies customer identity
   - May request additional identification
   - Security check (fraud prevention)

3. **Compilation**
   - Collect data from all systems
   - Remove third-party personal data (may redact)
   - Compile in accessible format

4. **Response**
   - Data provided within 30 days
   - Format: Electronic (CSV, PDF, etc.)
   - May extend 2 months for complex requests

5. **Redactions**
   - Legal professional privilege redacted
   - Third-party confidential data redacted
   - Trade secrets redacted
